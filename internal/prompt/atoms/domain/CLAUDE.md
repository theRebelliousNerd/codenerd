# domain/ - Project Domain Atoms

Project-specific and codebase context atoms.

## Files

| File | Purpose |
|------|---------|
| `project_context.yaml` | Project structure, conventions, tech stack |
| `codebase_style.yaml` | Code style, patterns, naming conventions |
| `encyclopedia.yaml` | Domain context reference |

## Usage

Domain atoms capture:
- Project architecture patterns
- Team conventions
- Technology stack details
- Domain-specific terminology

Selected via project detection or explicit configuration.

## Customization

For project-specific atoms beyond these templates, place custom atoms in `.nerd/agents/` which syncs to `.nerd/shards/`.
